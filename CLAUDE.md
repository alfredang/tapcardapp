## Build & run

The Xcode project is **generated by XcodeGen** from `project.yml` â€” do not hand-edit
`Tapcard.xcodeproj` (it is gitignored and regenerated).

```bash
xcodegen generate
xcodebuild -project Tapcard.xcodeproj -scheme Tapcard \
  -destination 'platform=iOS Simulator,name=iPhone 17' build CODE_SIGNING_ALLOWED=NO
```

**Important:** XcodeGen uses an explicit file list. After creating a new `.swift` file, run
`xcodegen generate` or the build won't see it ("cannot find X in scope").

### Conventions & gotchas
- **Backend contract**: `POST /api/mobile/onboard` takes the card fields + email, finds-or-creates
  the user, publishes the card, returns `{ card: { slug, url }, isNewAccount, password? }`.
  Field names in `BusinessCard` mirror the backend's `cardSchema`.
- **Swift 6 strict concurrency** is on. ViewModels are `@MainActor @Observable`. OCR runs off
  the main actor and crosses the boundary with `CGImage` (Sendable).
- **VisionKit delegate** (`VNDocumentCameraViewControllerDelegate`) is NOT `@MainActor` â€” the
  Coordinator stays nonisolated and hops to the main actor to deliver results.
- **Simulator has no camera** â†’ the review screen also supports manual entry ("Enter details
  manually") so the flow is testable without a camera.
